Description

Two pointers is a technique that is used to solve problems by iterating through the list with two pointers at different positions.

As we only consider singly linked list in this chapter, the two pointers are usually called fast and slow pointers.

The fast pointer moves at double speed or starts at a different position than the slow pointer.

While the fast pointer iterates through the list, the slow pointer tracks the position of desired node or element.

Things to take care of when using two pointers:

  • Check if a pointer is null before accessing its value or next node.
  • Consider special cases, especially when the list is empty or has only one node. This can usually be mitigated by using a dummy node pointing to the head of the list.

Examples

Type 1: Two pointers with different speed

In all the following examples, the fast pointer moves at double speed of the slow pointer.

Find the middle of a linked list

Given a linked list, find the middle node of the list.

Solution: the fast pointer reaches the end of the list when the slow pointer reaches the middle of the list.

Check if a linked list has a cycle

Given a linked list, determine if it has a cycle in it (which, if exists, starts from the tail and connects to some node in the list).

Solution: if there is a cycle in the linked list, the fast pointer will eventually meet the slow pointer. Otherwise, the fast pointer will reach the end of the list.

Find the start of a cycle in a linked list

The same problem as above, but this time we need to return the node where the cycle starts.

Solution: after the fast and slow pointers meet, we reset the slow pointer to the head of the list and move both pointers at the same speed. The node where the two pointers meet is the start of the cycle.

Type 2: Two pointers with different start positions

Find the k-th node from the end of a linked list

Given a linked list, find the k-th node from the end of the list.

Solution: we first move the fast pointer k steps forward. Then we move both pointers at the same speed until the fast pointer reaches the end of the list. The slow pointer is the k-th node from the end of the list.

Dummy node is used to handle the case where the list has less than k nodes.
